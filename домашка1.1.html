1. Git — система контроля и управления версиями файлов. GitHub — веб-сервис для размещения репозиториев и совместной разработки проектов. 2. add commit- фиксация изменений или запись изменений в репозиторий. push- отправка всех неотправленных коммитов на удалённый сервер репозитория. pull- получение последних изменений с удалённого сервера репозитория. branch- создавать новые ветки, удалять и переименовывать их/ checkout- используется для переключения веток и выгрузки их содержимого в рабочую директорию. merge- используется для слияния одной или нескольких веток в текущую. cherry-pick- создаёт новый коммит на основе только одного коммита, применив все его изменения и сообщение (будет создан новый коммит, дублирующий изменения из копируемого коммита). 3. - master является локальной ветвью - origin/master является удаленной ветвью (которая является локальной копией ветки с именем "master" на удаленном носителе с именем "origin" ) 4.
